Висновок з лабораторної роботи

Під час виконання лабораторної роботи розглянуто різні типи верстки HTML-документу, зокрема:

  • Таблична верстка - гумова та фіксована
  • Блокова верстка - гумова та фіксована
  • Верстка засобами CSS та Flexbox
Переваги різних типів верстки:
  • Таблична верстка - легкість освоєння та використання, візуально чіткий розподіл контенту.
  • Блокова верстка - менший обсяг коду у порівнянні з табличною версткою, широкі можливості для створення складних макетів та адаптивних дизайнів, зручність під час застосування стилів до елементів.
  • Верстка засобами CSS та Flexbox - дозволяє створювати надзвичайно гнучкі та адаптивні макети, надає зручні інструменти для вирівнювання та розподілу елементів, а також макети, створені цим типом верстки, гарно оптимізовані для сучасних браузерів.
Недоліки різних типів верстки:
  • Таблична верстка - дуже мала гнучкість, навіть маленькі зміни в одній ячейці можуть призвести до помилок у макеті, в різних браузерах макети можуть відображатися по-різному, потребує великої кількості коду.
  • Блокова верстка - складніша в освоєнні, ніж таблична верстка, у старих браузерах макети можуть відображатися неправильно.
  • Верстка засобами CSS та Flexbox - часто не підтримується старими браузерами.
Отже, підсумовуючи, таблична верстка є найстарішою та найменш зручною, потребує великого обсягу коду та не забезпечує адаптивності макету, і наразі майже не застосовується у великих проєктах. Блокова верстка забезпечує непогану адаптивність макетів та зручні методи для застосування стилів, проте є складнішою в освоєнні. Верстка методами Flexbox та CSS забезпечує відмінну адаптивність макетів та інструменти для керування розташуванням елементів, має майже відсутні недоліки, окрім відсутності підтримки у багатьох старих браузерах. Для більш детального ознайомлення з різними типами верстки було виконано завдання №2 та завдання №3, а для покращення макету веб-застосунку використано методи Flexbox та CSS.
